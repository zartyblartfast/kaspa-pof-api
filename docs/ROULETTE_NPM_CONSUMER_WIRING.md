# Roulette PoC published npm package wiring

This document records every repo point that makes `examples/roulette-poc/` consume the published `kaspa-pof-api@0.1.0-alpha.1` package instead of repo-local package source.

## Dependency source of truth

- `examples/roulette-poc/package.json`
  - Pins `kaspa-pof-api` to `0.1.0-alpha.1`.
- `examples/roulette-poc/package-lock.json`
  - Locks the published npm tarball resolved from `https://registry.npmjs.org/kaspa-pof-api/-/kaspa-pof-api-0.1.0-alpha.1.tgz`.
- `examples/roulette-poc/node_modules/kaspa-pof-api/`
  - Installed artifact used by the local PoC server and browser import map. It is generated by `npm install --prefix examples/roulette-poc` and should not be edited by hand.

## Browser wiring

- `examples/roulette-poc/app.js`
  - Imports browser-safe runtime primitives from:
    ```js
    import { deriveOutcome, verifyFairnessProof } from 'kaspa-pof-api/browser';
    ```
  - Keeps roulette-specific outcome derivation in the example consumer through `roulette-poc:number-v1`.
  - Calls `verifyFairnessProof()` in the browser after the server returns the portable proof bundle.

- `examples/roulette-poc/index.html`
  - Maps the browser subpath to the installed npm artifact:
    ```json
    "kaspa-pof-api/browser": "/examples/roulette-poc/node_modules/kaspa-pof-api/src/browser.mjs"
    ```
  - Must not map `kaspa-pof-api` or `kaspa-pof-api/browser` to `/src/browser.mjs` or any other repo-root `/src/*` package source path.

## Server wiring

- `examples/roulette-poc/server.cjs`
  - Uses the installed dependency for server-side package sanity checks:
    ```js
    require('kaspa-pof-api')
    ```
  - Does not require `../../src/index.cjs`.
  - Serves the installed package files under:
    ```text
    /examples/roulette-poc/node_modules/kaspa-pof-api/...
    ```
  - Does not serve repo-root `/src/*` package files.
  - Still only supplies round/evidence/proof-bundle plumbing. It must not return a trusted proof verdict that replaces browser package replay.

## Regression guards

- `test/roulette-runtime-consumer.test.mjs`
  - Fails if the browser app stops importing `kaspa-pof-api/browser`.
  - Fails if the import map returns to `/src/browser.mjs`.
  - Fails if the example dependency is not pinned to `kaspa-pof-api@0.1.0-alpha.1`.
  - Fails if the server returns to `../../src/index.cjs` or serves repo-root package source.
  - Fails if legacy HTTP proof-authority patterns return.

- `scripts/smoke.sh`
  - Checks the same package-consumer boundary during `npm run smoke`.
  - Excludes installed `node_modules` when scanning for stale legacy-reference text, because the published package contains public docs/reference material.

## Manual verification points

A correct browser load should request package modules from:

```text
/examples/roulette-poc/node_modules/kaspa-pof-api/src/browser.mjs
/examples/roulette-poc/node_modules/kaspa-pof-api/src/commitment.mjs
/examples/roulette-poc/node_modules/kaspa-pof-api/src/ledger.mjs
/examples/roulette-poc/node_modules/kaspa-pof-api/src/entropy.mjs
/examples/roulette-poc/node_modules/kaspa-pof-api/src/outcome.mjs
/examples/roulette-poc/node_modules/kaspa-pof-api/src/proof/verify.mjs
/examples/roulette-poc/node_modules/kaspa-pof-api/src/proof/root.mjs
/examples/roulette-poc/node_modules/kaspa-pof-api/src/networks/claim-levels.mjs
/examples/roulette-poc/node_modules/kaspa-pof-api/src/networks/kaspa-evidence.mjs
/examples/roulette-poc/node_modules/kaspa-pof-api/src/anchoring/evidence.mjs
```

A correct browser spin should end with:

```text
Claim level: tn10_future_entropy
Stage: verified
Operation: TN10 proof verified in browser
```

The old route `/src/browser.mjs` should not be served by the roulette PoC server.
